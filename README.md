## Три метода создания DLL

### 1. SimpleDll - Классический метод (__declspec)

**Особенности:**
- Использует `__declspec(dllexport)` для экспорта
- Использует `__declspec(dllimport)` для импорта
- Подходит для экспорта классов C++
- Требует подключения .lib файла при линковке

**Применение:** Стандартный способ для Windows DLL с C++ классами

### 2. QtPluginDll - Qt Plugin система

**Особенности:**
- Использует Qt метаобъектную систему
- Загружается динамически через QPluginLoader
- Не требует .lib файла
- Поддерживает интерфейсы через Q_INTERFACES

**Применение:** Расширяемые приложения, системы плагинов

### 3. ExternCDll - C-style интерфейс (extern "C")

**Особенности:**
- Экспорт функций в стиле C (без name mangling)
- Совместимость с другими языками программирования
- Простой и надежный интерфейс
- Не поддерживает экспорт классов

**Применение:** Межъязыковая совместимость, простые API

## Сборка проекта

### В Qt Creator:
1. Откройте файл `DllExample.pro`
2. Настройте kit (компилятор)
3. Нажмите "Build" -> "Build All"

### Из командной строки:
```bash
qmake DllExample.pro
make         # на Linux/Mac
nmake        # на Windows (MinGW: mingw32-make)
```

## Запуск

После сборки все файлы будут в папке `bin/`:
- SimpleDll.dll
- QtPluginDll.dll
- ExternCDll.dll
- MainApp.exe

Запустите `MainApp.exe` для тестирования всех трех DLL.

## Ключевые отличия методов

| Характеристика | SimpleDll | QtPluginDll | ExternCDll |
|----------------|-----------|-------------|------------|
| Экспорт классов | ✅ | ✅ | ❌ |
| Динамическая загрузка | ❌ | ✅ | ❌ |
| Нужен .lib файл | ✅ | ❌ | ✅ |
| C совместимость | ❌ | ❌ | ✅ |
| Qt зависимость | Минимальная | Полная | Нет |


## Примечания

- **Windows:** Все DLL-файлы должны находиться в той же папке, что и .exe, или в переменной PATH.
- **Linux:** Используйте .so вместо .dll и настройте переменную LD_LIBRARY_PATH.
- **macOS:** Используйте .dylib и настройте переменную DYLD_LIBRARY_PATH.
